## 2. Теоретичні відомості

### 2.1. Потоковий шифр XOR

Потоковий шифр — це симетричний шифр, у якому відкритий текст шифрується побітово за допомогою операції XOR з ключовим потоком (keystream). Основна властивість XOR, яка робить його придатним для шифрування:

```
A ⊕ K = C   (шифрування)
C ⊕ K = A   (дешифрування)
```

де `A` — відкритий текст, `K` — ключовий потік, `C` — шифротекст.

Оскільки `(A ⊕ K) ⊕ K = A`, операція дешифрування ідентична операції шифрування — достатньо повторно застосувати XOR з тим самим ключовим потоком.

### 2.2. Генератор Лемера (лінійний конгруентний генератор)

Для генерації ключового потоку використовується генератор Лемера з лабораторної роботи №2. Формула:

```
X(n+1) = (a * X(n)) mod m
```

де:
- `a = 16807` — множник
- `m = 2^31 - 1 = 2147483647` — модуль (просте число Мерсенна)
- `X(0)` — початкове значення (seed)

Цей генератор має повний період `m - 1`, що забезпечує достатню довжину псевдовипадкової послідовності.

### 2.3. Зміна ключа

Для підвищення криптостійкості реалізовано механізм зміни ключа (seed) після кожних 5 символів. Кожний блок шифрується власним ключовим потоком, що ускладнює криптоаналіз: навіть якщо зловмисник визначить ключ для одного блоку, це не дозволить розшифрувати інші блоки.

Формула зміни seed між блоками:

```
seed(i+1) = (seed(i) * 31 + pos + 1) mod 2147483646 + 1
```

## 3. Опис реалізації

### 3.1. Структура програми

Файл `mrk_main.py` містить 3 основні функції:

| Функція | Призначення |
|---|---|
| `generate_key_stream(seed, length)` | Генерує ключовий потік заданої довжини за допомогою генератора Лемера. Кожне згенероване число обрізається до діапазону 0–255 (`val % 256`) для XOR з байтами тексту. |
| `encrypt(plaintext, initial_seed, key_change_interval)` | Шифрує текст. Розбиває вхідні дані на блоки по 5 байт, для кожного блоку генерує новий ключовий потік зі зміненим seed, виконує побайтовий XOR. |
| `decrypt(ciphertext, initial_seed, key_change_interval)` | Дешифрує шифротекст. Алгоритм ідентичний шифруванню завдяки властивості XOR. |

### 3.2. Алгоритм шифрування (покроково)

1. Відкритий текст кодується у байти (UTF-8).
2. Байтовий масив розбивається на блоки по 5 байт.
3. Для кожного блоку:
   - Генерується ключовий потік довжиною, рівною розміру блоку, з поточного seed за допомогою генератора Лемера.
   - Кожен байт блоку XOR-иться з відповідним байтом ключового потоку.
   - Seed перераховується для наступного блоку.
4. Результат — масив зашифрованих байтів.

### 3.3. Алгоритм дешифрування

Повністю симетричний алгоритму шифрування. Оскільки `C ⊕ K = A`, та сама послідовність ключів відтворюється з того ж початкового seed, і XOR повертає оригінальні байти.

### 3.4. Код програми

```python
from LabTwo.NumericalGenerator import NumericalGenerator


def generate_key_stream(seed, length):
    gen = NumericalGenerator()
    modulus = 2147483647
    multi = 16807
    raw = gen.LemerGenerator(n=length, multi=multi, modulus=modulus, seed=seed)
    return [val % 256 for val in raw]


def encrypt(plaintext: str, initial_seed: int, key_change_interval: int = 5) -> bytes:
    data = plaintext.encode("utf-8")
    ciphertext = bytearray()
    current_seed = initial_seed
    pos = 0

    while pos < len(data):
        block_size = min(key_change_interval, len(data) - pos)
        key_stream = generate_key_stream(current_seed, block_size)

        for i in range(block_size):
            ciphertext.append(data[pos + i] ^ key_stream[i])

        current_seed = (current_seed * 31 + pos + 1) % 2147483646 + 1
        pos += block_size

    return bytes(ciphertext)


def decrypt(ciphertext: bytes, initial_seed: int, key_change_interval: int = 5) -> str:
    plaintext = bytearray()
    current_seed = initial_seed
    pos = 0

    while pos < len(ciphertext):
        block_size = min(key_change_interval, len(ciphertext) - pos)
        key_stream = generate_key_stream(current_seed, block_size)

        for i in range(block_size):
            plaintext.append(ciphertext[pos + i] ^ key_stream[i])

        current_seed = (current_seed * 31 + pos + 1) % 2147483646 + 1
        pos += block_size

    return plaintext.decode("utf-8")
```
